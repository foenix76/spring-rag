# 🛠️ 2026-04-12 HR 시스템 툴 호출 안정화 및 UI/UX 개선 회고

오늘 하루종일 형이랑 머리 싸매고 고생해서 드디어 HR 시스템의 핵심 기능을 괴물 급으로 단단하게 만들었어! AI가 말로만 때우던 버릇 고치고, UI도 훨씬 고급스러워진 대장정의 기록이야.

## 🎯 목표 (Goal)
- HR 일괄 메일 발송(`sendBulkResultEmail`) 툴 호출 성공률 100% 달성
- RAG 검색 결과와 툴 호출 카드의 공존 (데이터 유실 방지)
- 사용자 경험(UX) 고도화 (입력 제약 해제, 애니메이션 정제, 카드 노출 타이밍 조절)

## ⚔️ 고생과 사투의 기록 (The Struggle)

### 1. AI의 '말로만 때우기' 환각 (Tool Call Hallucination)
- **문제**: LLM이 "승인 카드를 띄워드렸습니다"라고 텍스트로는 말하지만, 실제 툴을 호출 안 해서 UI에 카드가 안 뜸.
- **원인**: 파라미터가 콤마로 구분된 긴 문자열이라 LLM이 부담을 느꼈고, 툴 설명이 약했음.
- **해결**: 
  - `HrActionTools.java`: `candidateIds` 타입을 `String`에서 `String[]`으로 변경하여 LLM이 구조적으로 인식하기 편하게 수정.
  - `@Tool` 설명에 **"말로만 하지 말고 무조건 직접 실행해라"**라고 강력한 경고 지시 주입.

### 2. RAG 검색 결과 증발 사건
- **문제**: 툴 호출은 성공했는데, 채팅창 본문에서 RAG로 찾은 지원자 정보(이름, 사유 등)가 싹 사라짐.
- **원인**: 시스템 프롬프트에 "과정만 부드럽게 설명하라"는 룰이 너무 빡세서 LLM이 지원자 목록을 스스로 검열해버림.
- **해결**: 
  - `RagService.java`: 시스템 프롬프트 3번 노드 수정. **"승인 카드를 띄울 때 반드시 RAG 기반 대상자 목록을 함께 나열할 것"**을 명시.

### 3. 시스템 메시지 RAG 검색 오작동
- **문제**: 사용자가 '취소' 버튼을 누르면 `[시스템 취소]` 메시지가 가는데, 이걸 RAG가 검색하려다 "정보 없음" 개소리를 시전함.
- **원인**: 시스템 제어용 메시지를 일반 질문으로 오인함.
- **해결**: 
  - `RagService.java`: 시스템 프롬프트에 6번 규칙 추가. `[시스템 승인/취소]` 문구 감지 시 부연 설명 없이 아주 짧게 대답하고 종료하도록 격리.

### 4. UI/UX 디테일 다듬기 (눈갱 방지 & 대기질 개선)
- **문제**: 전송 버튼 애니메이션이 너무 눈 아프고, AI가 대답할 때 형이 다음 질문을 입력조차 못 함.
- **해결**:
  - `index.html`: 
    - 전송 버튼에서 빙글빙글 도는 스피너 애니메이션 완전 삭제.
    - AI 스트리밍 중에도 입력창(`input`) 활성화 유지 (제출 버튼만 막음).
    - **가장 중요한 거**: 승인 카드가 텍스트 출력 도중에 튀어나와서 화면이 튀는 걸 막기 위해, **텍스트와 타이핑이 모두 끝나야 카드가 등장**하도록 지연 로직(`pendingApprovalToken`) 적용.
