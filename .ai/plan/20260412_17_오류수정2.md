# 17차: 화면 이동 로그 중복 출력 및 토큰 파싱 에러 수정, LLM 컨텍스트 요약 보강

## 1. RAG 기반 특정 인원 추출 시 (이유/근거 누락 현상 수정)
**이슈:** "인턴 경험자 3명에게 메일 발송해"와 같이 조건부 필터를 거칠 경우, 대상자 이름과 ID만 덩그러니 표출될 뿐 왜 해당 인원이 대상자가 되었는지에 대한 근거가 누락되어 있어 사용자 신뢰도가 떨어짐.
**개선 제안:** `RagService.java`의 시스템 프롬프트(System Prompt)를 강화하여 지원자 목록을 출력할 때 **한 줄 요약(근거)**을 반드시 포함하도록 Few-Shot 예제를 삽입.
```text
예시:
1. **최지민 (지원번호: 2026_034)** 대기업 인턴 경험 있음
2. **권수현 (지원번호: 2026_070)** 중소기업 인턴 경험 있음
```

## 2. 스트림 채팅 텍스트 중복 및 깨짐 (기형적인 Buffer 로직 제거)
**이슈:** 채팅창에 `[APPROVAL덕분에...` 형태의 깨진 문자열이 반복해서 나타나고 ID가 N/A로 바인딩되는 치명적 파싱 오류 발생.
**근본 원인:** 백엔드 `RagService`의 스트리밍 중간에 제어용 대괄호 `[` 가 한 번이라도 등장하면, 스트림 전송을 멈추고 `toolResultAccumulator`에 청크를 무한정 쌓아두다가 `]`를 만날 때 한 번에 방출하는 기형적인 필터 로직 때문. LLM이 `[참고 지원자 정보]에 의하면...` 와 같은 일반 문장을 생성하는 순간, 이 필터가 오동작하여 답변의 모든 텍스트를 흡수해버리고, 맨 마지막 토큰 발동 시 그동안 흡수한 모든 텍스트 덩어리를 통째로 중복 재전송하여 UI가 파괴됨.
**개선 조치:**
- 백엔드에 존재하던 "텍스트 지연 전송 및 필터링 로직"을 전면 삭제. LLM이 보내는 모든 텍스트 덩어리는 중간 필터링 없이 **무조건 즉시 프론트엔드로 전달**하도록 근본 분리.
- "백엔드는 단지 누적 텍스트에서 토큰이 등장했는지만 별개로 감지하여(Non-blocking) 토큰 이벤트만 이중 발송" 하도록 변경 (각 토큰당 스위치(AtomicBoolean) 적용).

## 3. NAVIGATE 무한 중복 및 노출 현상 제한
**이슈:** `화면을 이동합니다.`, `[NAVIGATE:/statistics]`, `채용 통계 보고서 화면으로 이동합니다.` 로그가 다중 중복 노출됨.
**개선 조치:**
- 프론트엔드 Vue 앱의 렌더러 로직(`renderText`)에 정규표현식 `replace(/\[NAVIGATE:[^\]]*\]/g, '')`을 추가.
- 백엔드에서 원본은 그대로 보내되 프론트에서 화면 렌더링 직전 실시간 다이나믹 마스킹(Dynamic Masking)을 수행하게 하여, `[NAVIGATE:/...]` 텍스트 자체가 화면에 보이지 않게 원천 차단.
- 백엔드 로그 삽입 로직에서 군더더기를 없애고 오직 토큰 이벤트 한 번만 프론트와 통신하도록 최적화.
