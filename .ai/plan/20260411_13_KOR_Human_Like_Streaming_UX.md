# 🚀 구현 계획서: 인간적인 리듬의 스트리밍 채팅 UX 고도화 (v1.0)

## 1. 개요 (Overview)
단순한 텍스트 출력을 넘어, **Buffer + Cursor 기반의 타이핑 효과**를 도입하여 AI가 실제로 생각하고 답변하는 듯한 리듬감 있는 UX를 구현합니다.

---

## 2. 핵심 변경 사항 (Key Changes)

### 2.1 CSS 감성 포인트 추가
- **`white-space: pre-wrap`**: 줄바꿈 및 공백 보존 (기존 적용 확인 및 강화).
- **커서 애니메이션**: 답변 끝에 깜빡이는 커서(`|`) 추가하여 생동감 부여.
- **Thinking 애니메이션**: AI가 고민 중일 때의 시각적 피드백 강화.

### 2.2 Vue3 상태 관리 (State Management)
- **`status`**: `thinking` -> `typing` -> `done` 상태 흐름 관리.
- **`buffer` & `cursor`**: 서버에서 받은 데이터를 버퍼에 쌓고, 커서로 한 글자씩 타이핑 출력.
- **`typingInterval`**: 일정한 리듬(15~30ms)으로 글자를 화면에 뿌려주는 엔진.

### 2.3 스트리밍 로직 리팩토링
- **서버 수신 (sse.js)**: 데이터를 받는 즉시 `buffer`에 누적.
- **화면 출력 (Typing Engine)**: 버퍼에 데이터가 있고 출력 커서가 뒤처져 있다면 리듬에 맞춰 한 글자씩 `text`에 추가.

---

## 3. 대상 파일 (Target Files)
- `src/main/resources/static/index.html`: CSS, Template, JS 로직 통합 수정.

---

## 4. 구현 단계 (Step-by-Step)

1.  **CSS 업데이트**: 커서 블링크 애니메이션 및 로딩 상태 스타일 정의.
2.  **Vue 상태 변수 확장**: `status`, `buffer`, `cursor` 등 추가.
3.  **타이핑 엔진 구현**: `setInterval`을 활용한 부드러운 출력 로직 작성.
4.  **메시지 수신부 연동**: `sse.js` 리스너에서 `buffer`에 데이터를 밀어넣는 구조로 변경.
5.  **검증**: 실제 답변 시 글자가 끊기지 않고 부드럽게 나오는지 확인.

---

## 5. 기대 효과 (Expected Outcome)
- 서버 응답이 조각조각(Chunk) 끊겨서 들어와도 화면에서는 **부드러운 타자기 효과** 유지.
- AI가 답변을 마친 후에도 커서가 끝까지 따라오며 **완성된 느낌** 제공.
- "생각 중..." 상태를 통해 유저에게 **심리적 안정감** 부여.
