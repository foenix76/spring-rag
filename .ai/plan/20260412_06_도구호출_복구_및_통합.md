# 멀티 에이전트 도구 호출(Tool Calling) 복구 계획

형이 말한 대로 "사용 가능 도구" 기능이 망가진 건, 요약 기능을 넣으면서 기존의 툴 등록 로직이 누락되었기 때문이야. 이를 복구하고 계단식 요약 및 멀티 에이전트 시스템과 완벽하게 통합할 계획을 세웠어.

## 주요 변경 사항

### [RagService](file:///home/foenix/workspace/spring-rag/src/main/java/com/example/korrag/service/RagService.java) [MODIFY]

- **도구 주입 및 등록**: 
    - `HrActionTools`와 `HrNavigationTools`를 생성자에서 주입받습니다.
    - `ChatClient.Builder`에 `.defaultTools()`를 사용하여 이 컴포넌트들을 등록합니다.
- **라우팅 에이전트 프롬프트 강화**: 
    - 라우팅 분석 시 도구 호출이 필요한 경우(`[ACTION]`)를 더 정확하게 포착하도록 지침을 추가합니다.
- **최종 답변 생성 및 도구 관리**:
    - 도구 호출 결과가 답변에 잘 녹아들도록 시스템 가이드를 보강합니다.

## 작업 순서

1.  `RagService.java` 파일에 도구 클래스 필드 추가 및 생성자 업데이트.
2.  `ChatClient` 빌드 시 도구 등록 로직 추가.
3.  `askStream` 메서드 내의 라우팅/최종 답변 프롬프트에 도구 사용 지침 추가.
4.  Maven 컴파일로 문법 오류 체크.
