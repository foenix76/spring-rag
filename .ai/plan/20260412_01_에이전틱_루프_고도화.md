# 🚀 에이전틱 루프 고도화 구현 계획서

형! 우리 RAG 시스템을 진짜 '천재 에이전트'로 만들기 위한 4단계 고도화 플랜이야. 단순히 묻고 답하는 수준을 넘어서, 스스로 생각하고, 허락받고, 요약하고, 역할을 나누는 구조로 탈바꿈해보자고!

## User Review Required

> [!IMPORTANT]
> **Human-in-the-Loop** 구현을 위해 프론트엔드(`index.html`)와 백엔드 간의 새로운 프로토콜(승인 요청 신호)이 추가됩니다. 답변 중간에 `[APPROVAL_REQUIRED:ACTION_NAME]` 같은 특수 토큰이 포함될 수 있습니다.

> [!WARNING]
> **Summarized Memory** 기능은 대화가 길어질 때 작동하며, 요약 과정에서 LLM 토큰이 추가로 소모됩니다.

## Proposed Changes

### 1. 🔍 Multi-Agent Coordination (역할 분담)
단일 프롬프트 대신, 내부적으로 단계를 나누어 처리하는 '가상 에이전트' 구조를 도입합니다.
*   **Search Agent**: 질문을 분석하고 최적의 검색 쿼리를 생성 (RagService 내 로직 분리)
*   **Analysis Agent**: 검색된 문서를 분석하여 핵심 내용 추출
*   **Execution Agent**: 최종 답변 작성 및 툴 호출 판단

### 2. 🧠 Summarized Memory (장기 기억 요약)
대화 이력이 너무 길어지는 것을 방지하고 핵심 맥락을 보존합니다.
*   **로직**: 최근 대화가 10개를 넘어가면, 이전 대화들을 하나로 요약하여 'Context Summary'로 저장합니다.
*   **적용**: `ChatMessageRepository`를 통해 과거 이력을 불러올 때, 요약본 + 최신 5개 메시지 조합으로 프롬프트를 구성합니다.

### 3. 🛡️ Self-Critique (자기 성찰 - Reflection)
AI가 답변을 내보내기 전, 스스로 모순이나 할루시네이션(환각)이 없는지 검토합니다.
*   **로직**: 최종 답변 생성 직후, 내부적으로 'Critique Prompt'를 실행하여 검색된 지원자 정보와 답변 내용이 일치하는지 확인합니다.
*   **처리**: 심각한 오류 발견 시 답변을 수정하거나 사용자에게 경고 문구를 함께 전달합니다.

### 4. 🤝 Human-in-the-Loop (메일 발송 승인)
민감한 작업인 **메일 발송** 실행 전 사용자의 최종 확인을 받습니다.
*   **Tool 수정**: `HrActionTools.sendResultEmail` 호출 시 즉시 발송 대신 `[APPROVAL_REQUIRED:EMAIL:candidateId:resultType]` 토큰이 포함된 메시지를 반환합니다.
*   **UI 연동**: 화면에 [메일 발송 승인] / [거절] 버튼을 띄우고, 승인 시에만 실제 메일 발송 로직(또는 성공 로그)이 수행되도록 구현합니다.
