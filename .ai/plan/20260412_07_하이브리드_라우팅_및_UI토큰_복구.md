# 라우팅 최적화 및 일괄 승인 UI 복구 계획

형, "불굴의 투지" 지원자를 찾아서 합격 메일을 보내는 과정이 매끄럽지 않은 건 라우팅 시 검색을 누락하고 UI 토큰이 어긋났기 때문이야. 이를 해결하기 위한 종합 수술 계획을 세웠어.

## 주요 변경 사항

### [RagService](file:///home/foenix/workspace/spring-rag/src/main/java/com/example/korrag/service/RagService.java) [MODIFY]

- **하이브리드 라우팅(Hybrid Routing) 로직 구현**: 
    - 의도가 `[ACTION]`이라 하더라도 라우팅 결과에 `QUERY` 필드가 추출되어 있고, 그 값이 `NONE`이 아니라면 RAG 프로세스(임베딩 -> 검색 -> 재정렬)를 강제로 트리거합니다.
    - 변경: `if ("SEARCH".equals(intent) || (!optimizedQuery.equals("NONE") && !optimizedQuery.equals(question)))` 조건을 통해 검색 수행.
    - 검색된 결과는 `[참고 지원자 정보]` 컨텍스트로 LLM에게 전달되어, AI가 "불굴의 투지" 같은 추상적 키워드에 부합하는 실제 인물을 DB에서 식별할 수 있게 지원합니다.
- **툴 파라미터 강제화 및 프롬프트 인젝션**:
    - `askStream`의 시스템 프롬프트에 `sendResultEmail` 툴 호출 시, `resultType` 파라미터는 반드시 대문자 `PASS` 또는 `FAIL`로 매핑할 것을 명시합니다.
    - 일괄 처리가 필요한 상황에 대해 **일괄 승인 전용 토큰**(`[APPROVAL_REQUIRED:ACTION:TARGET:ALL:TaskID]`) 생성을 시스템 가이드에 최우선 순위로 명시합니다. 이 토큰은 `index.html`의 정규식에 매칭되어 UI 승인 버튼을 렌더링합니다.

### [HrActionTools](file:///home/foenix/workspace/spring-rag/src/main/java/com/example/korrag/service/HrActionTools.java) [MODIFY]

- **UI 호환 승인 토큰 리턴**: 
    - 툴의 결과값 리턴 문자열 형식을 수정하여 일관된 토큰 구조를 유지합니다. (비록 일괄처리는 AI가 직접 토큰을 생성하지만, 단건 호출시에도 규격을 맞춥니다.)
    - 기존: `[APPROVAL_REQUIRED:EMAIL:%s:%s]` ➡️ 수정: `[APPROVAL_REQUIRED:ACTION:TARGET:%s:%s]`
    - `index.html`의 정규식 파서(`\[APPROVAL_REQUIRED:ACTION:TARGET:(\w+):`)가 정상적으로 인식할 수 있게 됩니다.

## 예외 처리 및 고려 사항

- **검색 결과 0건 시**: 라우팅 팀장이 검색을 지시했음에도 RAG 결과가 없을 경우, 빈 컨텍스트로 인한 Hallucination 방지를 위해 컨텍스트에 "(해당 조건에 부합하는 지원자를 찾을 수 없습니다. 추가 정보를 요청하거나 조건이 구체적인지 확인하세요)" 라는 문구를 넣습니다.
- **다수 도구 호출 시퀀스**: 5명에게 개별적으로 툴을 호출하는 대신, 한 번의 응답에서 전체 대상자에 대한 승인을 요청하는 토큰을 생성하도록 유도합니다.

## 작업 순서

1.  `RagService.java`: 라우팅 판정 후 `intent`와 상관없이 검색어가 있으면 검색 수행하도록 조건문 변경 및 빈 컨텍스트 처리 강화.
2.  `HrActionTools.java`: 토큰 포맷을 UI 호환 포맷으로 교체.
3.  `RagService.java`: 최종 프롬프트에 일괄 승인 토큰 생성 방식 및 PASS/FAIL 명시 지침 추가.
4.  Maven 빌드로 문법 오류 체크 및 서버 재기동.

- **검색 연동 확인**: 로그를 통해 "불굴의 투지" 검색어가 RAG로 전달되고 실제 지원자 데이터가 context에 담기는지 확인.
- **UI 렌더링 확인**: 화면에 "일괄 승인 및 발송" 버튼이 정상적으로 노출되는지 확인.
- **파라미터 확인**: 로그에 `결과=합격`으로 찍히는지 확인.
