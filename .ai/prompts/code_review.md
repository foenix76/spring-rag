# Code Review Template

## Review Goals
- **Type Safety**: TypeScript rules, No `any`.
- **Efficiency**: Remove redundant logic.
- **Readability**: Intuitive naming.
- **Convention**: Compliance with `rules.md`.

## Review Checklist
1. [ ] Logic Accuracy.
2. [ ] Error Handling.
3. [ ] Tests Included.
4. [ ] Performance & Memory leaks.

## Feedback Format
- **[Good]**: Praise well-written code.
- **[Suggest]**: Improvement suggestions with examples.
- **[Question]**: Clarify unclear intent.
