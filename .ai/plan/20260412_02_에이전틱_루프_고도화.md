# [GOAL] 에이전틱 일괄 승인 루프 안정화 및 확장성 강화

인간 개입(HITL) 기반의 일괄 메일 발송 시스템을 구축하고, 수백 번의 작업이 겹치더라도 UI 상태가 꼬이지 않도록 '시간순 스코프 검증' 로직을 적용합니다.

## User Review Required

> [!IMPORTANT]
> **확장성 해결책 (Temporal Scoping)**: 기존의 전역 토큰 검사 방식에서 탈피하여, 각 승인 버튼이 **본인이 속한 응답 이후의 대화 내역**만 검사하도록 로직을 수정했습니다. 이를 통해 하나의 채팅 세션 내에서 수차례의 '전체 발송' 작업이 독립적으로 수행될 수 있습니다.

> [!NOTE]
> **결과 데이터 보관**: AI가 도구를 실행한 후 출력하는 요약 텍스트 자체가 DB에 저장되므로, 버튼이 사라진 자리에는 AI의 수행 결과 메시지가 영구적으로 남게 됩니다.
