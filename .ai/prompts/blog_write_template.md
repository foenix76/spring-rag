## 🚀 초대형 분량 및 퀄리티 확보 지침 (Ultimate Expansion Strategy)
글의 분량을 압도적으로 늘리기 위해 다음 지침을 **모든 섹션**에서 반드시 준수하세요.

1. **하위 섹션의 심층화 (Deep Hierarchical Structure)**:
   - 모든 `##` 소제목 아래에는 반드시 최소 3개 이상의 `###` 하위 소제목을 생성하세요.
   - 각 하위 섹션 안에서도 필요하다면 `####`를 사용하여 내용을 더 쪼개세요.
   - 각 문단은 최소 5줄 이상의 풍부한 설명으로 구성하세요.

2. **3단계 타겟별 맞춤 설명 (Multi-Tier Use Cases)**:
   - 모든 핵심 개념마다 다음 세 그룹을 위한 전용 사례를 상세히 묘사하세요.
     1) **초등학생**: 일상 속 놀이나 장난감에 비유한 아주 쉬운 설명.
     2) **일반인/기획자**: 비즈니스 관점과 실생활 편의성 관점의 설명.
     3) **전문 개발자**: 실제 코드 로직, API 구조, 시스템 아키텍처 관점의 깊이 있는 설명.

3. **비교 도표 및 심층 분석 (Comparative Matrix)**:
   - 주요 기술마다 라이벌 기술(예: RAG vs Fine-tuning)과의 상세 비교 표(Table)를 최소 2개 이상 포함하세요.
   - 표에는 최소 5개 이상의 비교 항목(비용, 속도, 정확도, 난이도, 유연성 등)을 넣으세요.

4. **기술적 원리의 시각화 묘사 (Mechanism Visualization)**:
   - "작동한다"라고 하지 말고, 데이터가 어떤 관문을 거쳐 어떻게 변하고 어디에 저장되는지 '그림을 그리듯' 문장으로 상세히 묘사하세요.

5. **10년 뒤 미래 시나리오 (Futuristic Vision)**:
   - 해당 기술이 보편화된 10년 뒤의 일상을 아주 구체적인 에피소드로 풀어내며 마무리하세요.

## 🎨 이미지 프롬프트 생성 전략 (Target Model Optimized)
사용자가 선택할 모델에 따라 아래 두 가지 스타일의 프롬프트를 모두 작성하세요.

### 1. Juggernaut-XL 스타일 (초실사/고성능)
- **목표**: 실제 사진과 구분이 안 될 정도의 극사실주의.
- **필수 키워드**: `Photorealistic`, `8k resolution`, `Cinematic lighting`, `Highly detailed skin texture`, `Sharp focus`, `Masterpiece`.
- **인물 묘사**: `(highly detailed faces:1.2)`, `sparkling expressive eyes`. 얼굴 디테일 확보를 위해 `Medium shot` 혹은 `Portrait` 구도를 사용하세요.
- **Negative 유도**: 프롬프트 자체에 `Clean anatomy`, `Perfect eyes` 등 긍정적 유도를 포함하세요.

### 2. Nanobanana 스타일 (예술적/일러스트/광속)
- **목표**: 깔끔하고 세련된 테크 일러스트레이션 혹은 3D 렌더링 느낌.
- **필수 키워드**: `Clean vector art`, `Minimalist tech illustration`, `Vibrant colors`, `Soft shadows`, `High-quality digital art`.
- **특징**: 복잡한 질감보다는 명확한 형태와 상징적인 오브젝트 위주로 묘사하세요.
