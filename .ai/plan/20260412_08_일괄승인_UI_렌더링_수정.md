# 일괄 승인 UI 렌더링 및 프롬프트 수정 계획

## 현상 분석
형이 말한 "데이터가 한 줄 찍히고 버튼 생겼다 사라지고" 현상은 다음과 같은 복합적인 이유로 발생했어.
1. **타이핑 중 토큰 노출 (삭제 현상의 원인)**: UI에서 `[APPROVAL_REQUIRED...` 토큰이 완성(`]`)되기 전까지는 정규식에 매칭되지 않아서 사용자 화면에 텍스트로 보이다가, `]`가 타이핑되는 순간 정규식에 잡혀 텍스트가 사라지는(삭제되는) 현상이 발생.
2. **반복 출력**: LLM이 5명의 대상자에 대해 툴을 호출하거나, 토큰을 대상자마다 각각 생성하여 출력하는 바람에 버튼이 여러 번 평가됨.
3. **Task ID 파싱 오류**: `[APPROVAL_REQUIRED:ACTION:TARGET:ALL:TASK_ID]` 구조에서 Task ID를 파싱하는 정규식이 `TARGET:(\w+):`로 되어 있어서 `ALL`을 Task ID로 잘못 파싱하고 있었음.

## 주요 변경 사항

### 1. `src/main/resources/static/index.html` [MODIFY]
- **부분 토큰 숨김 처리 (`renderMessage`)**:
  - 기존: `text.replace(/\[APPROVAL_REQUIRED:.*?\]/gi, '')`
  - 수정: `text.replace(/\[APPROVAL_REQUIRED[^\]]*\]?/gi, '')` 
  - (ACTION_COMPLETED 동일 적용)
  - 이렇게 하면 괄호가 닫히기 전 타이핑 중인 상태에서도 화면에 노출되지 않아 깔끔해짐.
- **Task ID 정규식 수정 (`getTaskId`)**:
  - 기존: `/\[APPROVAL_REQUIRED:ACTION:TARGET:(\w+):/i`
  - 수정: `/\[APPROVAL_REQUIRED:ACTION:TARGET:ALL:(\w+)\]?/i`
  - `ALL` 다음의 실제 Task ID를 정확히 추출하도록 변경.

### 2. `src/main/java/com/example/korrag/service/RagService.java` [MODIFY]
- **시스템 프롬프트 명확화**:
  - LLM이 대상자별로 토큰을 뱉거나 불필요하게 툴을 호출하지 않도록 지침을 매우 강력하게 수정.
  - "일괄 처리가 필요한 경우(예: 다수 메일 발송), **절대 툴을 먼저 호출하지 말고** 먼저 대상자 목록을 모두 나열한 뒤, **응답 맨 마지막에 단 한 번만** `[APPROVAL_REQUIRED:ACTION:TARGET:ALL:TaskID]` 토큰을 생성하여 승인을 요청하세요. 승인이 확인된 후에만 툴을 호출하세요."
